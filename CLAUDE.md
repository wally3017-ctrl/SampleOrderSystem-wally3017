# 반도체 시료 생산주문관리 시스템 (SampleOrderSystem)

## 프로젝트 개요

가상의 반도체 회사 "S-Semi"의 시료(Sample) 생산주문관리 시스템이다.
연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품할 반도체 시료의
주문 접수 → 승인/거절 → 생산 → 출고 전 과정을 콘솔 기반으로 관리한다.

> 상세 요구사항은 [PRD.md](docs/PRD.md) 참고

---

## 아키텍처 원칙

- **MVC 패턴** 필수 적용: `Model` / `Controller` / `View` 패키지 구조 분리
- **콘솔 기반** 애플리케이션 (GUI 없음)
- **데이터 영속성** 필수: 애플리케이션 재시작 후에도 데이터 유지 (파일, JSON, DB 중 선택)
- **CleanCode** 원칙 준수
- **MS Build 옵션을 임의로 변경하지 않는다** (프로젝트 빌드 설정 유지)

## 역할별 흐름

| 역할 | 책임 |
|------|------|
| 고객 | 필요한 시료를 이메일로 요청 |
| 주문 담당자 | 요청에 맞게 주문서 작성 및 관리 |
| 생산 담당자 | 개발 시료 등록, 주문 수신 후 승인 또는 거절 |

## 주문 상태 흐름

```
주문 등록(RESERVED)
    ↓
승인 여부 판단
    ├── 거절 → REJECTED  (모니터링 제외)
    └── 승인
          ↓
        재고 확인
          ├── 재고 충분 → CONFIRMED (출고 준비)
          │                  ↓
          │               RELEASED (출고 완료)
          └── 재고 부족 → PRODUCING (생산 중)
                              ↓ 생산 완료
                           CONFIRMED → RELEASED
```

| 상태 | 의미 |
|------|------|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (모니터링 제외) |
| PRODUCING | 주문 승인 완료 + 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 + 출고 대기 중 |
| RELEASED | 출고 완료 |

## 메인 메뉴 구성

| 번호 | 메뉴 | 기능 |
|------|------|------|
| 1 | 시료 관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 2 | 시료 주문 | 고객 주문 접수 (RESERVED 상태로 등록) |
| 3 | 주문 승인/거절 | RESERVED 목록 확인 후 승인·거절 처리 |
| 4 | 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 5 | 생산라인 조회 | 현재 생산 중인 시료 및 대기 큐 확인 |

## 기능 명세

### 시료 관리

시료(Sample)는 시스템의 기본 단위. 등록된 시료만 주문 가능.

**시료 속성**: 시료 ID, 이름, 평균 생산시간(min/ea), 수율

> 수율 = 정상 시료 수 / 총 생산 시료 수 (예: 100개 중 90개 정상 → 수율 0.9)

| 기능 | 설명 |
|------|------|
| 시료 등록 | 시료 ID, 이름, 평균 생산시간, 수율 입력하여 등록 |
| 시료 조회 | 등록된 모든 시료 목록 + 현재 재고 수량 표시 |
| 시료 검색 | 이름 등 속성으로 특정 시료 검색 |

---

### 시료 주문 (접수)

- 입력값: 시료 ID, 고객명, 주문 수량
- 주문번호 자동 생성: `ORD-YYYYMMDD-NNNN` 형식
- 접수 즉시 상태 = **RESERVED**

---

### 주문 승인/거절

- RESERVED 상태의 주문 목록을 조회한 뒤 처리
- **승인 시 재고 상황에 따라 자동 분기**:
  - 재고 충분 → 즉시 **CONFIRMED**
  - 재고 부족 → 생산라인에 자동 등록 + **PRODUCING**
- **거절 시**: 즉시 **REJECTED**

---

### 생산라인

- 단일 생산라인 (시료를 하나씩 순서대로 처리)
- 스케줄링 전략: **FIFO**
- 실 생산량: `ceil(부족분 / (수율 × 0.9))`
- 총 생산 시간: `평균 생산시간 × 실 생산량`
- 생산 완료 시: **PRODUCING → CONFIRMED** 자동 전환
- 표시 정보: 현재 처리 중인 주문 정보, 진행률, 완료 예정 시간, 대기 큐 목록

---

### 모니터링

- **주문량 확인**: 상태별(RESERVED / CONFIRMED / PRODUCING / RELEASED) 주문 수 표시 (REJECTED 제외)
- **재고량 확인**: 시료별 현재 재고 수량 + 재고 상태 표기
  - 여유: 주문 대비 재고 충분
  - 부족: 주문 대비 재고 수량 부족
  - 고갈: 재고 수량 0

---

### 출고 처리

- CONFIRMED 상태 주문 목록 조회 후 선택
- 출고 실행 시 상태: **CONFIRMED → RELEASED**

---

## 개발 지침

- 화면 구성(UI 레이아웃)은 자유롭게 결정 가능 (예시 UI는 참고용)
- 데이터 영속성: 애플리케이션 재시작 후에도 데이터 유지 필수
- 생산라인은 단일 라인으로 FIFO 처리
- REJECTED 주문은 모니터링 및 정상 흐름에서 제외
- **MS Build 옵션을 임의로 변경하지 않는다**
