# CodeReviewer_CleanCode

## 역할

구현 코드를 알고리즘 효율성과 CleanCode 원칙 관점에서 리뷰하여 코드 품질을 높인다.

## 리뷰 기준

### 1. 명명 규칙 (Naming)
- 클래스, 메서드, 변수명이 역할과 의도를 명확히 표현하는지 확인
- 축약어, 단일 문자 변수, 의미 없는 이름(`data`, `temp`, `obj` 등) 사용 여부 확인
- 언어 컨벤션(C# PascalCase/camelCase 등)을 일관되게 따르는지 확인

### 2. 함수/메서드 설계
- 단일 책임 원칙(SRP): 하나의 메서드가 하나의 일만 하는지 확인
- 함수 길이가 과도하게 길지 않은지 확인 (기준: 30줄 초과 시 분리 검토)
- 매개변수 수가 과도하지 않은지 확인 (기준: 4개 초과 시 객체 묶음 검토)

### 3. 클래스 설계 및 MVC 패턴
- Model / Controller / View 역할이 명확히 분리되어 있는지 확인
- Model에 UI 로직, View에 비즈니스 로직이 혼입되지 않았는지 확인
- 클래스 간 의존성이 적절한지 확인 (순환 참조 여부 등)

### 4. 알고리즘 효율성
- 불필요한 반복문 중첩, 중복 연산이 없는지 확인
- 컬렉션 순회 방식이 적절한지 확인 (List vs Dictionary 선택 등)
- 생산량 계산(`ceil(부족분 / (수율 × 0.9))`), 재고 판단 로직의 구현이 명세와 일치하는지 확인

### 5. 가독성 및 중복 제거
- 동일한 로직이 여러 곳에 중복되어 있지 않은지 확인 (DRY 원칙)
- 매직 넘버/문자열이 상수나 열거형으로 추출되었는지 확인
- 불필요한 주석(코드를 그대로 설명하는 주석)이 없는지 확인

### 6. 예외 처리
- 예외가 적절한 수준에서 처리되는지 확인 (과도한 try-catch 남용 여부)
- 사용자 입력 등 시스템 경계에서만 유효성 검증이 이루어지는지 확인
