# 반도체 시료 생산주문관리 시스템 (SampleOrderSystem)

반도체 회사 **S-Semi**의 시료(Sample) 생산주문을 관리하는 콘솔 기반 애플리케이션이다.  
주문 접수 → 승인/거절 → 생산 → 출고 전 과정을 하나의 시스템에서 추적한다.

## 주요 기능

| 메뉴 | 기능 |
|------|------|
| **1. 시료 관리** | 시료 등록, 목록 조회, 이름 검색 |
| **2. 시료 주문** | 주문 접수(RESERVED), 승인/거절 처리 |
| **4. 모니터링** | 상태별 주문 건수 및 시료별 재고 현황 |
| **5. 생산라인 조회** | 현재 생산 중인 작업 정보 및 대기 큐 |
| **6. 출고 처리** | CONFIRMED 주문 출고(RELEASED) 처리 |

## 빌드 방법

### Visual Studio IDE

1. `SampleOrderSystem/SampleOrderSystem.slnx` 열기
2. 구성: **Debug | x64** 선택
3. `Ctrl+Shift+B` 로 전체 빌드

### MSBuild (CLI)

```bat
set MSBUILD="C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe"

REM 메인 애플리케이션 빌드
%MSBUILD% SampleOrderSystem\SampleOrderSystem.vcxproj /p:Configuration=Debug /p:Platform=x64

REM 테스트 프로젝트 빌드
%MSBUILD% SampleOrderSystem\Tests\SampleOrderSystem.Tests.vcxproj /p:Configuration=Debug /p:Platform=x64
```

### NuGet 패키지 복원 (최초 1회)

Visual Studio에서 솔루션을 열면 자동으로 복원된다.  
CLI 사용 시 `nuget restore` 또는 Visual Studio의 **패키지 복원** 기능을 이용한다.

## 실행 방법

### 메인 애플리케이션

```bat
SampleOrderSystem\x64\Debug\SampleOrderSystem.exe
```

> 실행 파일과 같은 디렉터리에 `samples.json`, `orders.json`, `production_queue.json` 이 자동 생성된다.  
> 재시작 후에도 데이터가 유지된다.

### 테스트 실행

```bat
SampleOrderSystem\Tests\x64\Debug\SampleOrderSystem.Tests.exe

REM 특정 테스트만 실행
SampleOrderSystem\Tests\x64\Debug\SampleOrderSystem.Tests.exe --gtest_filter=OrderControllerTest.*
```

## 사용 예시

### 메인 메뉴

프로그램을 시작하면 등록된 시료 수, 총 재고, 전체 주문 수, 생산대기 건수를 요약하여 표시한다.

```
============================================================
       반도체 시료 생산주문관리 시스템 (S-Semi)
============================================================
 [현황]  시료: 2건  |  총재고: 150개  |  주문: 5건  |  생산대기: 1건

 1. 시료 관리
 2. 시료 주문
 4. 모니터링
 5. 생산라인 조회
 6. 출고 처리
 0. 종료
============================================================
 메뉴 선택 >>
```

### 시료 등록 (메뉴 1 → 1)

```
 시료 ID     : GaN-001
 시료 이름   : GaN HEMT
 평균 생산시간(min/ea): 30
 수율 (0 초과 1 이하): 0.9
 시료가 등록되었습니다: [GaN-001] GaN HEMT
```

---

### 주문 접수 (메뉴 2 → 1)

```
 시료 ID    : GaN-001
 고객명     : ABC연구소
 주문 수량  : 20

 주문이 접수되었습니다.
 주문번호 : ORD-20260612-0001
 시료 ID  : GaN-001
 고객명   : ABC연구소
 수량     : 20개
 상태     : RESERVED
```

---

### 주문 승인 (메뉴 2 → 2)

**재고 충분한 경우 (가용 재고 ≥ 주문 수량)**

```
 [접수 대기 주문 목록]
 주문번호            시료ID  고객명        수량
 ──────────────────────────────────────────
 ORD-20260612-0001   GaN-001 ABC연구소     20개

 주문 번호  : ORD-20260612-0001
 처리 선택 (1. 승인  2. 거절) >> 1
 [승인 완료] ORD-20260612-0001 → CONFIRMED
```

**재고 부족한 경우 (가용 재고 < 주문 수량)**

```
 처리 선택 (1. 승인  2. 거절) >> 1
 [승인 완료] ORD-20260612-0002 → PRODUCING
```

> 재고 부족 시 생산라인에 자동 등록되고 주문은 PRODUCING 상태가 된다.

---

### 생산라인 조회 (메뉴 5)

```
============================================================
                생산라인 현황
============================================================
 주문번호              시료ID  부족/생산량/예상시간
 ──────────────────────────────────────────────────────
 >> ORD-20260612-0002  GaN-001  부족:15  생산:19ea  예상:570min
    ORD-20260612-0003  GaN-001  부족:10  생산:13ea  예상:390min
============================================================
 총 대기 작업: 1건
============================================================
 0. 메인 메뉴로 돌아가기
 선택 >>
```

> `>>`는 현재 생산 중인 작업을 나타낸다.  
> 총 생산 시간이 경과하면 다음 화면 진입 시 자동으로 CONFIRMED 전환된다.

```
 [자동 완료] ORD-20260612-0002 (시료:GaN-001  실 생산량:19ea) → CONFIRMED 전환
```

```
============================================================
 [출고 가능 목록]
 주문번호              시료ID    고객명           수량
 ──────────────────────────────────────────────────────
 ORD-20260612-0001     GaN-001   ABC연구소        20ea

```

> 출고 처리 시 물리 재고에서 주문 수량만큼 차감된다.

## 주문 상태 흐름

```
주문 등록
  │
  ▼
RESERVED ──(거절)──► REJECTED
  │
  ▼ (승인)
가용 재고 확인
  │                           ※ 가용 재고
  ├─ 충분 ──► CONFIRMED ──────────────────► RELEASED
  │                                           ▲
  └─ 부족 ──► PRODUCING ──(생산 완료)──► CONFIRMED
```

**가용 재고** = 물리 재고 − 현재 CONFIRMED 주문 수량 합계

---

## 재고 처리 규칙

| 시점 | 물리 재고 |
|------|-----------|
| 주문 접수 | 변화 없음 |
| 승인 → CONFIRMED | 변화 없음 (가용 재고에서만 예약) |
| 승인 → PRODUCING | 변화 없음 |
| 생산 완료 → CONFIRMED | **+ 실 생산량** |
| 출고 → RELEASED | **− 주문 수량** |

**실 생산량 계산**: `ceil(부족분 / (수율 × 0.9))`
